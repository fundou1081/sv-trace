# ADR-001: 使用 pyslang 作为解析引擎

## 状态

> **状态**: Accepted
> **日期**: 2026-04-17

## 摘要

选择 pyslang 作为 SV Trace 库的 SystemVerilog 解析引擎，提供 AST 解析、语义分析和 Python 绑定能力。

---

## 1. 背景

### 当前状态
需要为 SV Trace 库选择一个 SystemVerilog 解析引擎。

### 问题陈述
- 需要解析完整的 SystemVerilog 语法
- 需要 Python binding 集成
- 需要支持大型项目（数千个文件）
- 需要活跃的维护

### 约束条件
- Python 3.11+ 环境
- macOS/Linux 兼容

---

## 2. 决策

### 核心决策
使用 [pyslang](https://github.com/MikePopoloski/slang) 作为解析引擎。

### 决策理由
- **优势**: 
  - Python binding 成熟稳定
  - 解析鲁棒，支持不完整代码
  - 活跃维护（MIT 许可证）
  - 支持 class/constraint/assertion 等高级特性
- **权衡**:
  - 某些 API 封装可能导致性能损耗

### 考虑的替代方案

#### 方案 A: sv-lang/slang C++ 库
- **描述**: 原生 C++ 库
- **未选择原因**: 需要自行封装 Python binding

#### 方案 B: Verilator
- **描述**: 开源 Verilog 编译器
- **未选择原因**: 不支持 Python binding，主要用于仿真

---

## 3. 后果

### 积极影响
- 直接使用 Python API 开发上层应用
- 复用 slang 测试用例
- 降低开发门槛

### 消极影响 / 风险
- 依赖外部 PyPI 包
- API 稳定性需要关注版本发布
