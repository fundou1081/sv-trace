# Team Communication Log

跨团队沟通记录

## 功能

1. 记录跨团队的关键讨论
2. 追踪决策和变更
3. 记录问题升级
4. 追踪任务分配

## 记录类型

### 1. 决策记录 (Decision)

```markdown
## DEC-2026-001

**日期**: 2026-04-27
**会议**: UART模块设计评审
**参与方**: 设计, 验证, 后端

**议题**: FIFO深度选择

**选项**:
1. 16字节 (当前设计)
2. 32字节 (验证建议)
3. 64字节 (后端建议)

**决策**: 32字节

**理由**:
- 16字节在115200波特率下可能不够
- 32字节面积增加<5%
- 64字节面积增加>15%

**影响**:
- 设计: 需要修改FIFO深度参数
- 验证: 更新测试场景
- 后端: 更新综合脚本

**签字**:
- 设计: 张三
- 验证: 李四
- 后端: 王五
```

### 2. 问题升级 (Escalation)

```markdown
## ESC-2026-001

**日期**: 2026-04-27
**级别**: P1
**升级方**: 验证
**接收方**: 设计

**问题**: TX路径setup violation

**描述**:
在115200波特率最高频率下，TX路径有setup violation，
WNS = -0.5ns

**影响**:
- 可能导致数据错误
- 需要在tapeout前修复

**要求**:
- 设计在4/30前提供解决方案
- 验证协助分析

**回复期限**: 2026-04-28
```

### 3. 变更通知 (Change)

```markdown
## CHG-2026-001

**日期**: 2026-04-27
**发起方**: 设计
**通知方**: 验证, 后端, 软件

**变更类型**: 接口变更

**变更描述**:
信号名变更:
- tx_data → tx_payload
- rx_data → rx_payload

**原因**:
统一命名规范，避免与内部信号混淆

**影响评估**:
- 验证: 需要更新TB信号连接
- 后端: 无影响
- 软件: 需要更新寄存器定义

**实施计划**:
1. 4/27: RTL更新
2. 4/28: 验证更新
3. 4/29: 软件更新

**截止日期**: 2026-04-29
```

### 4. 澄清记录 (Clarification)

```markdown
## CLF-2026-001

**日期**: 2026-04-27
**提问方**: 验证
**回答方**: 设计

**问题**:
Spec中描述"波特率误差<1%"，这个1%是指:
A) 发射端波特率误差
B) 接收端采样误差容忍度
C) 端到端误差

**回答**:
是接收端采样误差容忍度。TX实际波特率必须精确，
但RX需要能容忍±1%的误差。

**确认人**: 设计(张三)
```

## 分类索引

### 按类型
- DEC: 设计决策
- ESC: 问题升级
- CHG: 变更通知
- CLF: 澄清记录
- TASK: 任务分配

### 按模块
- UART控制器
- FIFO
- 时钟管理
- 复位逻辑

### 按状态
- open: 待处理
- in_progress: 进行中
- resolved: 已解决
- closed: 已关闭

## 使用流程

```
1. 新讨论/决策/问题
   ↓
2. 创建记录 (按模板)
   ↓
3. 分配相关方
   ↓
4. 追踪行动项
   ↓
5. 定期回顾
   ↓
6. 关闭/归档
```

## 定期检查

| 频率 | 内容 |
|------|------|
| 每日 | 检查open项 |
| 每周 | 回顾本周决策 |
| 每月 | 清理已关闭项 |

## 格式要求

1. 每个记录有唯一ID
2. 包含日期和参与者
3. 行动项有owner和截止日期
4. 状态清晰可追踪
