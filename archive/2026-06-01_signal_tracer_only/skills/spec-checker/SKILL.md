# Spec Checker Skill

规格文档完整性检查

## 功能

检查Spec文档是否包含以下必要章节：

### 1. 基本信息
- [ ] 模块名称和版本
- [ ] 功能描述
- [ ] 应用场景

### 2. 接口定义
- [ ] 端口列表（名称、方向、位宽）
- [ ] 时序图或时序描述
- [ ] 协议说明

### 3. 功能规格
- [ ] 状态机定义（如有）
- [ ] 数据通路描述
- [ ] 控制逻辑说明

### 4. 寄存器定义
- [ ] 寄存器地址映射
- [ ] 字段定义
- [ ] 复位值

### 5. 时序规格
- [ ] 时钟域划分
- [ ] 时序约束
- [ ] CDC处理

### 6. 边界条件
- [ ] 工作范围
- [ ] 异常处理
- [ ] 错误检测

### 7. 验证计划
- [ ] 验证要点
- [ ] 覆盖率目标

## 使用方法

### 自动检查
```
用户提供Spec文档内容
助手检查完整性并报告缺失项
```

### 检查规则

```
1. 读取Spec文档
2. 检查必要章节是否存在
3. 识别模糊或歧义描述
4. 报告问题和建议
```

## 输出格式

```markdown
# Spec完整性检查报告

## 检查结果: ❌ 不通过

## 缺失章节:
- [ ] CDC处理方案
- [ ] 寄存器定义

## 模糊描述:
- "数据应该及时传输" - 请明确时间要求
- "必要时可关闭" - 请明确条件

## 建议:
1. 添加CDC协议章节
2. 完善寄存器定义表格
```

## 检查项清单

| 章节 | 必要性 | 检查点 |
|------|--------|--------|
| 模块概述 | 必须 | 名称/版本/功能 |
| 接口定义 | 必须 | 端口/时序/协议 |
| 功能描述 | 必须 | 状态机/数据流 |
| 寄存器 | 推荐 | 地址/字段/复位 |
| 时序 | 必须 | 时钟/约束/CDC |
| 边界条件 | 推荐 | 范围/异常/错误 |
| 验证计划 | 推荐 | 验证点/覆盖 |

## 示例

**输入**: Spec片段
```
模块功能: UART控制器
支持9600-115200波特率
```

**输出**:
```
❌ Spec完整性不足

缺失必要项:
- 缺少端口定义
- 缺少寄存器定义
- 缺少时序约束

建议补充:
1. 添加端口列表
2. 定义CTRL/STAT寄存器
3. 添加波特率精度要求
```
