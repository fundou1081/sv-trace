# Spec Corner Case Detector

规格边界条件识别

## 功能

从规格文档中识别和提取：
1. 数值边界条件
2. 异常场景
3. 边界测试点
4. 潜在问题

## 边界条件类型

### 1. 数值边界

| 类型 | 示例 | 边界值 |
|------|------|--------|
| 范围 | 波特率9600-115200 | 9600, 115200 |
| 计数 | FIFO深度16 | 0, 1, 15, 16, 17 |
| 位宽 | 8位数据 | 0x00, 0x7F, 0x80, 0xFF |
| 超时 | 超时100ms | 0, 1, 99, 100, 101 |

### 2. 时序边界

| 类型 | 示例 | 边界 |
|------|------|------|
| 建立时间 | t_su | 最小值 |
| 保持时间 | t_h | 最小值 |
| 延迟 | t_d | 最小/最大值 |

### 3. 协议边界

| 场景 | 边界 |
|------|------|
| 空闲 | 0周期空闲, 1周期空闲 |
| 帧 | 最小帧, 最大帧 |
| 突发 | 最小突发, 最大突发 |

## 识别规则

### 数值提取
```
关键词: 最大/最小/不超过/至少/不超过
模式: "波特率不超过115200" → max=115200
模式: "响应时间至少100ns" → min=100ns
```

### 异常场景
```
关键词: 如果/当/除非/异常/错误/故障
模式: "如果FIFO满" → 溢出场景
模式: "除非复位" → 复位边界
```

## 输入示例

```markdown
规格摘要:
- 波特率: 9600-115200
- 数据位: 5,6,7,8
- 停止位: 1,2
- FIFO深度: 16字节
- 超时: 100ms ±10ms
- 传输: 8N1标准格式
```

## 输出示例

```markdown
# 边界条件清单

## 1. 数值边界

### 波特率
| 边界 | 值 | 测试点 |
|------|-----|--------|
| 最小 | 9600 | 9600时精度 |
| 最大 | 115200 | 115200时精度 |
| 精度 | ±1% | 中间值测试 |

### 数据位宽
- 最小: 5bit
- 最大: 8bit
- 边界: 4bit(非法), 9bit(非法)

### FIFO
- 深度: 16字节
- 边界: 0, 1, 15, 16, 17(溢出)

### 超时
- 范围: 90ms-110ms
- 边界: 89ms, 90ms, 110ms, 111ms

## 2. 异常场景

### FIFO溢出
条件: 写入第17字节时FIFO已满
处理: 丢弃数据或阻塞

### 波特率不匹配
条件: 收发波特率不一致
影响: 数据错误

### 复位时序
条件: 传输过程中复位
处理: 停止传输，清空FIFO

## 3. 测试建议

| 优先级 | 测试场景 |
|--------|----------|
| P0 | 115200/8N1标准传输 |
| P0 | FIFO满溢出 |
| P1 | 9600最低波特率 |
| P1 | 超时边界90ms |
| P2 | 5位数据位 |
| P2 | 连续复位 |

## 4. 潜在问题

1. 波特率误差在临界值(±1%)时行为?
2. FIFO满时write行为未定义?
3. 超时检测的时钟精度要求?
```

## 提取流程

```
1. 解析Spec文本
2. 识别数值范围
3. 提取关键词
4. 生成边界表
5. 识别异常场景
6. 生成测试建议
```
