# Debug Knowledge Base

Debug经验知识库

## 功能

1. 记录常见RTL问题及解决方案
2. 提供Debug方法论指导
3. 积累团队经验

## 问题分类

### 1. CDC问题

| 问题 | 症状 | 根因 | 解决方案 |
|------|------|------|----------|
| CDC亚稳态 | 随机错误 | 同步器不足 | 增加2FF同步器 |
| CDC数据错误 | 跨时钟域数据错乱 | 无握手 | 添加握手协议 |
| CDC丢失数据 | 数据偶尔不见 | 快到慢 | 使用FIFO或握手 |
| CDC重复数据 | 同数据收到多次 | 慢到快 | 添加确认机制 |

### 2. 时序问题

| 问题 | 症状 | 根因 | 解决方案 |
|------|------|------|----------|
| Setup violation | WNS < 0 | 路径太长 | 流水线/优化逻辑 |
| Hold violation | WHN < 0 | Skew太大 | 添加delay |
| Metastability | 输出不稳定 | 亚稳态 | 同步器 |

### 3. 复位问题

| 问题 | 症状 | 根因 | 解决方案 |
|------|------|------|----------|
| 复位无效 | 复位后状态随机 | 复位时序 | 同步释放 |
| 复位冲突 | 多复位源冲突 | 逻辑错误 | 优先级编码 |
| 复位蔓延慢 | 部分模块不复位 | reset_tree问题 | 树平衡 |

### 4. FIFO问题

| 问题 | 症状 | 根因 | 解决方案 |
|------|------|------|----------|
| FIFO溢出 | 数据丢失 | 满时仍写 | 满标志检测 |
| FIFO下溢 | 读空数据 | 空时仍读 | 空标志检测 |
| FIFO不满 | 空不满 | 读写指针错 | Gray码编码 |

### 5. 状态机问题

| 问题 | 症状 | 根因 | 解决方案 |
|------|------|------|----------|
| 死锁 | 卡在某个状态 | 转移条件错误 | 检查所有转移 |
| default缺失 | latch推断 | 未覆盖所有条件 | 添加default |
| 异步状态 | 状态随机 | 状态未同步 | 同步输入 |

### 6. X传播问题

| 问题 | 症状 | 根因 | 解决方案 |
|------|------|------|----------|
| X输出 | 信号出现X | 多驱动/未初始化 | 检查驱动/加复位 |
| X保持 | X不消失 | 反馈回路 | 断开回路加复位 |
| X比较 | X参与运算 | 条件含X | 避免X参与运算 |

## Debug方法论

### 1. 二分法Debug

```
1. 确定问题范围
   ├─── 上次正常? → 最近的变更
   │           YES → 变更引入
   │           NO → 之前就存在
   │
   ├─── 随机/确定? 
   │           随机 → 时序/CDC问题
   │           确定 → 逻辑问题
   │
   └─── 局部/全局?
               局部 → 子模块问题
               全局 → 系统问题
```

### 2. 隔离法

```
1. 禁用部分模块
2. 逐个使能
3. 定位问题模块
```

### 3. 对比法

```
1. 对比仿真与真实行为
2. 对比RTL与参考模型
3. 对比不同seed结果
```

## Debug检查清单

### CDC检查
- [ ] 所有异步输入有同步器?
- [ ] 同步器级数 >= 2?
- [ ] 多bit使用握手/FIFO?
- [ ] 复位同步释放?

### 时序检查
- [ ] WNS > 0?
- [ ] 无transition violation?
- [ ] clock tree balanced?

### 复位检查
- [ ] 所有寄存器有复位?
- [ ] 复位同步释放?
- [ ] 复位与时钟关系正确?

### 状态机检查
- [ ] 有default状态?
- [ ] 所有转移有条件?
- [ ] 转移覆盖完整?

## 常见pattern

### 1. 2FF同步器
```systemverilog
reg meta1, meta2;
always @(posedge clk or negedge rst_n)
    if (!rst_n)
        {meta2, meta1} <= 2'b0;
    else
        {meta2, meta1} <= {meta1, async_in};
```

### 2. Gray码FIFO指针
```systemverilog
// Gray码编码
assign wptr_gray = wptr ^ (wptr >> 1);
```

### 3. 握手同步器
```systemverilog
// req/ack协议
reg req_sync, ack_sync;
always @(posedge dst_clk)
    {ack_sync, req_sync} <= {req_sync, src_req};
```

## 经验总结

### 当遇到X时
1. 检查是否有未初始化的寄存器
2. 检查是否有多个驱动
3. 检查是否有feedback回路
4. 检查是否有异步复位

### 当遇到随机错误时
1. 可能是CDC问题
2. 可能是时序violation
3. 可能是复位问题
4. 尝试固定随机seed

### 当遇到性能问题时
1. 检查关键路径
2. 检查fanout
3. 检查门控效率
4. 考虑流水线
