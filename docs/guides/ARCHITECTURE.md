## Parser Foundation v1.0

### 设计原则

1. **AST 优先** - 所有解析器使用 pyslang AST 遍历,无正则表达式
2. **原子化** - 每个解析器专注单一语法类别
3. **可组合** - 基于现有 parser 可派生新 parser

## 依赖关系说明

### 高依赖 (trace/)

`trace/` 模块直接操作 AST 进行信号追踪,63% 的文件直接依赖 parse:

- `driver.py` - 驱动追踪
- `load.py` - 负载追踪
- `dataflow.py` - 数据流分析
- `connection.py` - 连接追踪
- `controlflow.py` - 控制流分析

### 中依赖 (lint/)

`lint/` 模块需要源码进行模式匹配,75% 依赖 parse:

- `linter.py` - 代码风格检查
- `code_quality.py` - 代码质量评分
- `syntax_check.py` - 语法检查

### 低依赖 (debug/verify/query/dft)

这些模块有自己的封装或使用其他方式获取信息:

- `constraint_parser_v2.py` - 使用 SVParser 封装
- 其他模块通过 trace/ 的结果进行二次分析
