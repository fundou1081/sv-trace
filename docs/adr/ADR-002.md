# ADR-002: DriverTracer 与 LoadTracer 设计

## 状态

> **状态**: Accepted
> **日期**: 2026-04-17

## 摘要

实现信号追踪核心模块，包括 DriverTracer（追踪驱动源）和 LoadTracer（追踪加载点）。

---

## 1. 背景

需要实现信号的驱动/加载追踪功能，用于：
- debug 过程中追踪信号的 drive/load 关系
- 数据流分析
- 控制流分析

### 约束条件
- 基于 pyslang AST
- 性能和准确性平衡

---

## 2. 决策

### 架构设计

```
SVParser
    ├── DriverTracer
    │   └── find_driver(signal_name) → List[Driver]
    │       ├── _find_in_module()
    │       ├── _extract_continuous_assign()  # assign xxx = yyy
    │       ├── _extract_always_block()         # always_ff @(posedge clk)
    │       └── _find_assignment_in_stmt()       # 递归查找
    │
    └── LoadTracer
        └── find_load(signal_name) → List[Load]
            ├── _find_in_module()
            ├── _extract_continuous_assign()  # assign 中的 RHS 引用
            ├── _extract_always_load()    # always 块中的引用
            └── _find_load_in_stmt()     # 递归查找
```

### 核心实现

- **类型识别**: 通过 `str(type(node))` 包含关键字判断
- **变量名提取**: 使用 `str(var).strip()`
- **位置信息**: 使用 `location.offset`

### 性能考虑
- 延迟解析：只在需要时遍历 AST
- 字符串匹配用于变量名（简化实现）

---

## 3. 后果

### 积极影响
- 支持 assign/always_ff 驱动追踪
- 支持连续赋值中的加载点追踪
- 可扩展支持更多块类型

### 已知限制
