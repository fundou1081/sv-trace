# ADR-012: 模块依赖分析

> **状态**: Planning
> **日期**: 2026-04-19

---

## 需求概述

分析模块间的依赖关系:
1. 模块被哪些模块引用
2. 模块引用了哪些子模块
3. 依赖层级图
4. 循环依赖检测

---

## 算法方案

### Option A: 实例化追溯

```
1. 遍历所有模块实例化语句
2. 提取实例名和模块类型
3. 构建 实例→模块 映射
4. 反向构建 依赖图
```

| 优点 | 缺点 |
|------|------|
| 简单直接 | 只能找到直接实例化 |
| 准确 (基于 AST) | 无法处理动态实例化 |
| 快速 O(n) | |

### Option B: 接口分析

```
1. 分析模块端口类型
2. 匹配端口连接关系
3. 通过信号连接推断依赖
```

| 优点 | 缺点 |
|------|------|
| 可以找到隐式依赖 | 实现复杂 |
| 更精确 | 端口重名会误判 |

### Option C: 完整依赖链

```
1. 实例化追溯 + 接口分析
2. 加上参数传递链
3. 加上跨模块信号追踪
```

| 优点 | 缺点 |
|------|------|
| 最完整 | 最复杂 |
| 可检测循环依赖 | 性能较差 |

### Option D: 层级分析

```
1. 按层级组织模块
2. 同层级模块可能有并行关系
3. 父子依赖明确
```

| 优点 | 缺点 |
|------|------|
| 适合 SoC 分析 | 不适合平面图 |
| 结构清晰 | 需要层次信息 |

---

## 推荐方案

**Option A (实例化追溯)**: 快速实现，获取核心依赖图

**Option C (完整依赖链)**: 最终目标，支持循环检测

---

## 输出格式

```python
@dataclass
class ModuleDependency:
    module: str
    instances: List[InstanceInfo]  # 当前模块的实例
    depended_by: List[str]  # 依赖此模块的模块
    depends_on: List[str]  # 此模块依赖的模块

@dataclass
class DependencyGraph:
    modules: Dict[str, ModuleDependency]
    cycles: List[List[str]]  # 循环依赖
    root_modules: List[str]  # 顶层模块 (无依赖)
    leaf_modules: List[str]  # 叶子模块 (无子模块)
```
