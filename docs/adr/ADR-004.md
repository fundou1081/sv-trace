# ADR-004: 数据流深度追踪与 Case 语句处理

## 状态

> **状态**: Accepted
> **日期**: 2026-04-18

## 摘要

实现数据流深度追踪功能 (DataPathAnalyzer)，从 input 追踪到 output，支持 always_comb/always_ff 内的赋值语句，特别是 case 语句和 begin...end 块的正确处理。

---

## 1. 背景

原 DriverTracer 存在以下问题：
- 无法处理 always_comb/always_ff 内的赋值
- case 语句中的驱动无法识别（使用了错误属性 `item.body`）
- begin...end 块未被处理
- 模块迭代逻辑对 CompilationUnit 处理不正确

---

## 2. 决策

### 架构设计

```
DataPathAnalyzer (新增)
├── find_driver()           # 查找信号驱动源
├── analyze(signal)         # 深度分析数据流
└── visualize()             # 可视化输出

DriverTracer (修复)
├── _find_all_drivers()    # 修复模块迭代
├── _visit_member()        # 修复 SyntaxNode 遍历
├── _extract_always_block() # 提取 always 块
├── _find_assignment_in_stmt() # 修复 case/block 处理
└── _detect_always_type()  # 检测 always 类型
```

### 核心修复

#### 1. 模块迭代修复
```python
# 之前：只处理直接的 ModuleDeclaration
if 'ModuleDeclaration' not in str(type(root)):
    continue

# 之后：正确处理 CompilationUnit
modules = []
if 'ModuleDeclaration' in str(type(root)):
    modules = [root]
elif hasattr(root, 'members') and root.members:
    modules = [root.members[i] for i in range(len(root.members)) 
              if 'ModuleDeclaration' in str(type(root.members[i]))]
```

#### 2. SyntaxNode 遍历修复
```python
# 之前：无法遍历 SyntaxNode
if isinstance(child, list):
    for c in child:
        drivers.extend(self._visit_member(c, signal_name))
else:
    drivers.extend(self._visit_member(child, signal_name))

# 之后：正确处理 SyntaxNode
if isinstance(child, list):
    for c in child:
        drivers.extend(self._visit_member(c, signal_name))
elif hasattr(child, "__len__") and hasattr(child, "__getitem__"):
    for idx in range(len(child)):
        drivers.extend(self._visit_member(child[idx], signal_name))
else:
    drivers.extend(self._visit_member(child, signal_name))
```

#### 3. Case 语句修复
```python
# 之前：使用错误属性
for item in stmt.items:
    if hasattr(item, 'body') and item.body:
        self._find_assignment_in_stmt(item.body, ...)

# 之后：使用正确属性 clause
for idx in range(len(stmt.items)):
    item = stmt.items[idx]
    if hasattr(item, 'clause') and item.clause:
        self._find_assignment_in_stmt(item.clause, ...)
```

#### 4. BlockStatement 处理
```python
# 新增：处理 begin...end 块
elif 'Block' in type_name:
    if hasattr(stmt, 'items') and stmt.items:
        for idx in range(len(stmt.items)):
            self._find_assignment_in_stmt(stmt.items[idx], ...)
```

## 7. 后续修复 (2026-04-18 晚)

### If/Else 修复

pyslang 的 `ConditionalStatementSyntax` 使用以下属性：
- `predicate` - 条件表达式
- `statement` - if 分支
- `elseClause` - else 分支（类型为 `ElseClauseSyntax`）
- `elseClause.clause` - else 的实际内容（可能是另一个 ConditionalStatement 或 BlockStatement）

### 代码变更

```python
# 之前：使用不存在的属性
if hasattr(stmt, 'ifBody') and stmt.ifBody:
    ...
if hasattr(stmt, 'elseBody') and stmt.elseBody:
    ...

# 之后：正确使用 pyslang 属性
elif 'ConditionalStatement' in type_name:
    # if 分支
    if hasattr(stmt, 'statement') and stmt.statement:
        self._find_assignment_in_stmt(stmt.statement, ...)
    # else 分支 - 通过 elseClause.clause 访问
    if hasattr(stmt, 'elseClause') and stmt.elseClause:
        else_clause = stmt.elseClause
        if hasattr(else_clause, 'clause') and else_clause.clause:
            self._find_assignment_in_stmt(else_clause.clause, ...)
```

### 测试结果

```
=== Simple If/Else ===
Drivers for r: 2
  - kind=ALWAYS_FF, source=r = a
  - kind=ALWAYS_FF, source=r = b

=== Nested If/Else-If/Else ===
Drivers for r: 3
  - kind=ALWAYS_FF, source=r = a
  - kind=ALWAYS_FF, source=r = b
  - kind=ALWAYS_FF, source=r = c
```

### Always_ff 修复

pyslang 对 always_ff 的处理：
- `ProceduralBlock.statement` 是 `TimingControlStatementSyntax`
- 包含 `timingControl` 属性 (@(posedge clk))
- 实际语句在 `TimingControlStatementSyntax.statement` 中

```python
# 之前：直接处理 statement
if 'TimingControl' in str(type(stmt)):
    if hasattr(stmt, 'statement'):
        self._process_expression_stmt(stmt.statement, ...)

# 之后：处理 TimingControlStatement
if 'TimingControlStatement' in str(type(stmt)):
    if hasattr(stmt, 'statement') and stmt.statement:
        self._find_assignment_in_stmt(stmt.statement, ...)
```

| 测试 | 状态 | Drivers |
|------|------|---------|

| Nested If/Else-If/Else | ✅ | 3 |
| Case Statement | ✅ | 3 |
| Always_ff | ✅ | 2 |
| Mixed If/Case | ✅ | 4 |

Driver 类型现在正确识别：ALWAYS_COMB vs ALWAYS_FF

## 8. 循环语句支持 (2026-04-18 晚)

### 问题

pyslang 的循环语句使用不同的属性名：
- ForLoopStatement → 用 `.statement` 而不是 `.body`
- WhileLoop (LoopStatement) → 用 `.statement`
- DoWhileStatement → 用 `.statement`
- ForeachLoopStatement → 用 `.statement`

### 代码修复

```python
# ForLoopStatement
elif 'ForLoop' in type_name:
    if hasattr(stmt, 'statement') and stmt.statement:
        self._find_assignment_in_stmt(stmt.statement, ...)

# WhileLoop
elif 'LoopStatement' in type_name or 'WhileLoop' in type_name:
    if hasattr(stmt, 'statement') and stmt.statement:
        ...

# DoWhile
elif 'DoWhile' in type_name:
    ...

# Foreach
elif 'ForeachLoop' in type_name:
    ...
```

### 测试结果

| 语句类型 | 状态 | Drivers |
|----------|------|---------|
| For Loop | ✅ | 2 |
| While Loop | ✅ | 2 |
| Do-While | ✅ | 1 |
| Foreach | ✅ | 2 |

## 9. 流水线追踪 (2026-04-18 晚)

### 功能

`DataPathAnalyzer.analyze(signal)` 返回：
- **Pipeline Stages** - 识别 always_ff 寄存器链的阶段
- **Data Flow** - 从 input 到 output 的完整数据流
- **Dependencies** - 每个信号的驱动来源和类型

### 算法

1. 构建信号依赖图
2. 识别 ALWAYS_FF 驱动的信号
3. 递归计算 stage 索引：stage(sig) = stage(upstream) + 1
4. 按 stage 分组形成流水线阶段

### 测试结果

| 测试 | 状态 | Stages | Data Flow |
|------|------|--------|-----------|
| 3-Stage Pipeline | ✅ | 3 | din → s1 → s2 → dout |
| With Enable | ✅ | 3 | din → s1 → s2 → dout |
| With Reset | ✅ | 3 | din → s1 → s2 → dout |
| With Combo | ✅ | 1 | a+b → add → mul → result |

### API

```python
from trace.datapath import DataPathAnalyzer

analyzer = DataPathAnalyzer(parser)
dp = analyzer.analyze('dout')
print(dp.visualize())

# 或分析整个模块
dp = analyzer.analyze_module()
print(dp.visualize())
```
