# ADR-020: FSM深度分析与状态编码建议

## 状态: 已接受

## 背景

FSMAnalyzer 已完成基础功能(状态提取、跳转提取、死锁检测、不可达状态检测)。根据 TODO_V2.md，需要增强以下功能:

1. **状态编码建议** - 根据状态数推荐编码方式
2. **复杂度评分增强** - 完善评分模型
3. **验证视角增强** - 状态覆盖率、验证计划生成
4. **SVA属性自动生成** - 用于形式验证

## 决策

### 1. 状态编码推荐算法

```python
def recommend_encoding(state_count: int) -> dict:
    if state_count <= 2:
        return {"encoding": "binary", "bits": 1, "reason": "2状态用binary最省资源"}
    elif state_count <= 4:
        return {"encoding": "binary", "bits": 2, "reason": "4状态用binary"}
    elif state_count <= 8:
        return {"encoding": "gray", "bits": 3, "reason": "Gray编码减少亚稳态"}
    elif state_count <= 16:
        return {"encoding": "one_hot", "bits": state_count, "reason": "one-hot高速"}
    else:
        return {"encoding": "one_hot", "bits": state_count, "reason": "多状态建议拆分"}
```

### 2. 复杂度评分模型

```python
# 基础复杂度 = 状态数 × 平均跳转数
complexity = state_count * avg_transitions

# 考虑额外因素
# - 嵌套深度 (nested case/if)
# - 状态位宽
# - 跨时钟域

complexity_score = (
    state_count * avg_transitions + 
    nested_depth * 10 + 
    bit_width_factor
)
```

### 3. 验证计划生成

基于状态机自动生成验证点:
- 每个状态的进入/退出
- 每条跳转路径
- 边界条件(上电、复位)

### 4. SVA属性生成

```systemverilog
// 状态机属性示例
sequence state_transition;
    state == IDLE ##1 state == RUN;
endsequence

property check_idle_to_run;
    @(posedge clk) disable iff (!rst_n)
    state == IDLE |-> ##1 state == RUN;
endproperty
```
