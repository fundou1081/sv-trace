# ADR-003: 数据流与控制流应用层设计

## 状态

> **状态**: Accepted
> **日期**: 2026-04-18

## 摘要

实现基于底层 sv-trace 库的应用层，包括层级路径解析、端口方向识别、代码片段提取等功能。

---

## 1. 背景

sv-trace 底层库已完成核心追踪功能（DriverTracer/LoadTracer），需要构建应用层来提供：
- 跨模块层级路径解析（如 `top.cpu.u_alu.result`）
- 端口方向识别（input/output/inout）
- 代码片段提取（用于展示驱动/负载代码）
- 完整数据流追踪

---

## 2. 决策

### 架构设计

```
sv-trace 应用层
├── query/hierarchy/
│   ├── resolver.py       # HierarchicalResolver
│   │   ├── resolve_signal(path)    # 解析层级路径
│   │   ├── get_all_instances()      # 获取所有实例+端口
│   │   ├── _get_module_ports()      # 获取模块端口+方向
│   │   └── _find_child_module()     # 追踪子模块
│   │
│   └── code_extractor.py # CodeExtractor, SourceViewer
│       ├── get_snippet()           # 提取代码片段
│       ├── get_driver_snippet()    # 提取驱动代码
│       └── show_signal()           # 展示信号完整信息
│
└── trace/
    └── dataflow.py       # DataFlowTracer (已有)
        ├── find_flow()              # 找驱动+负载
        └── find_flow_chain()        # 递归追踪数据流链
```

### 核心实现

#### 层级路径解析
- **路径分解**: `HierarchicalPath` 类解析 `top.cpu.u_alu.result`
- **模块查找**: `_find_module_by_name()` 通过 `header.name.value` 匹配
- **成员访问**: pyslang 的 `ModuleDeclarationSyntax` 使用 `members` 而非 `body`
- **实例追踪**: `HierarchyInstantiation` 的模块类型在 `type.value` 中

#### 端口方向识别
- **端口列表**: `header.ports[1]` 是 `SeparatedList` 包含所有端口
- **端口类型**: `ImplicitAnsiPortSyntax` 表示 ANSI 风格端口
- **方向获取**: `item.header.direction.kind` 对应 `TokenKind.InputKeyword/OutputKeyword`
- **缓存机制**: `_port_cache` 避免重复解析

#### 代码片段提取
- **定位方式**: 使用 offset 而非行号（pyslang 使用 offset）
- **上下文**: 支持提取周围 N 行代码

## 3. 功能清单

| 功能 | 状态 | 说明 |
|------|------|------|
| 简单信号解析 | ✅ | `resolve_signal("data")` |
| 层级信号解析 | ✅ | `resolve_signal("top.u_cpu.reg_data")` |
| 模块端口提取 | ✅ | 提取 input/output/inout |
| 实例端口连接 | ✅ | 端口 -> 连接信号 + 方向 |
| 代码片段提取 | ✅ | 按 offset 提取 |
| 驱动代码获取 | ✅ | `get_driver_snippet()` |
| 负载代码获取 | ✅ | `get_load_snippet()` |
| 完整数据流 | ✅ | `DataFlowTracer.find_flow()` |

## 4. 测试结果

```python
# 层级解析
resolver.resolve_signal("top.u_cpu.reg_data")
# → {'signal': 'reg_data', 'module': 'cpu', 'path': 'top.u_cpu.reg_data'}

# 端口方向
instances = resolver.get_all_instances()
# [{'instance_name': 'u_cpu', 'ports': [
#     {'port': 'clk', 'direction': 'input', 'connected_to': 'clk'},
#     {'port': 'result', 'direction': 'output', 'connected_to': 'result'}
# ]}]
```

---

## 5. 后果

### 积极影响
- 支持跨模块信号追踪
- 可识别端口方向（input/output）
- 提取实际代码用于 debug

### 已知限制
- 位选信号追踪待完善
- 多层次实例（>2层）需进一步测试
- 参数化模块实例化未支持

### 待办
- 完善多层级实例追踪
- 添加数据流可视化功能
- 支持更复杂的表达式解析
